PreTrainedTokenNameFinder

  • Evaluation
  • Output

This processor wraps the evaluation from OpenNLP Name Finder. The Name Finder can detect named entities and numbers in text. To be able to detect entities the Name Finder needs a model. The model is dependent on the language and entity type it was trained for. The OpenNLP projects offers a number of pre-trained name finder models which are trained on various freely available corpora. They can be downloaded at our model download page. To find names in raw text the text must be segmented into tokens and sentences. It is important that the tokenization for the training data and the input text is identical. It loads its model from a file path.

Evaluation

PreTrainedTokenNameFinder runs against:

  • A tokens list, as produced by a TrainableTokenizer or a PreTrainedTokenizer.

Input is read directly from flow file attributes, allowing to chain the processors.

Since all data required by the processor lies in the attributes, the processor will not open flow file content.

Output

PreTrainedTokenNameFinder will enrich flow file attributes with

  • nlp.token.name.finder.names.list: the names list
  • nlp.token.name.finder.names.span: the names spans
