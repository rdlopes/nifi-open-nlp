OpenNLP Document Categorizer

This processor wraps the evaluation from OpenNLP Document Categorizer . It creates its model from training data provided to the processor via configuration.

OpenNLP Document Categorizer classifies a text into trained categories, allowing for semantic parsing and sentiment analysis.

Training

OpenNLP Document Categorizer takes training data in a proprietary format. There must be one entry per line, category being written first, a TAB or SPACE character then the training text for that category separated bby SPACEs. 

  • TRAINABLE_TRAINING_FILE_PATH
    pointing at an existing training file path. 
  • TRAINABLE_TRAINING_DATA
    containing the raw data to train the processor.

Sample training data:

GMDecrease Major acquisitions that have a lower gross margin than the existing network also \
           had a negative impact on the overall gross margin, but it should improve following \
           the implementation of its integration strategies.
GMIncrease The upward movement of gross margin resulted from amounts pursuant to adjustments \
           to obligations towards dealers.

Input Data

As to work properly, processor requires the following attributes to be available in the flow file:

  • nlp.sentence.detector.sentences.list: A tokens list, built by the means of a TrainableTokenizer or a PreTrainedTokenizer.

Since all data required by the processor lies in the attributes, the processor will not open flow file content.

Output Data

OpenNLP Chunker chunks text into smaller meaningful parts, and writes its evaluation results to two attributes:

  • nlp.pos.tagger.tags.list: The list of chunks evaluated.
  • nlp.pos.tagger.tags.list: The spans for each chunk found.
