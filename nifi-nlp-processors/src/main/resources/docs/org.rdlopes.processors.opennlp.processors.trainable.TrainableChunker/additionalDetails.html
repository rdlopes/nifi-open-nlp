OpenNLP Chunker

This processor wraps the evaluation from OpenNLP Chunker . It creates its model from training data provided to the processor via configuration.

OpenNLP Chunker splits text into syntactically correlated parts for sentence analysis.

Training

OpenNLP Chunker takes training data in CoNLL2000 format. You can configure training data for the processor by using one of the two properties:

  • TRAINABLE_TRAINING_FILE_PATH pointing at an existing training file path.
  • TRAINABLE_TRAINING_DATA containing the raw data to train the processor.

Sample training data:

He        PRP  B-NP

Input Data

As to work properly, processor requires the following attributes to be available in the flow file:

  • nlp.tokenizer.tokens.list: A tokens list, built by the means of a TrainableTokenizer or a PreTrainedTokenizer.
  • nlp.pos.tagger.tags.list: A tags list, built by the means of a TrainablePOSTagger or a PreTrainedPOSTagger.

Since all data required by the processor lies in the attributes, the processor will not open flow file content.

Output Data

OpenNLP Chunker chunks text into smaller meaningful parts, and writes its evaluation results to two attributes:

  • nlp.pos.tagger.tags.list: The list of chunks evaluated.
  • nlp.pos.tagger.tags.list: The spans for each chunk found.
